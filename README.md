# ダーツログ

カウントアップ / クリケットカウントアップのスコアを記録・管理する個人用アプリ（PWA）。

## 機能

- **カウントアップ**（8R×3投、ファットブル基準。設定でセパレートに変更可。3投入力→「ラウンド確定」で次へ。1ラウンド100点以上は LOW TON として結果画面に表示）
- **クリケットカウントアップ**（ラウンド固定ターゲット: R1=20, R2=19, R3=18, R4=17, R5=16, R6=15, R7=ブル, R8=15〜20とブル全対象。入力はクリケットのマーク表記 ／・✕・⊗ で表示。MPR 計測）
- **フリースロー**（スコア入力なし。アワードカウンターとメモだけの練習画面）
- **ダーツライブ記録の取り込み**（カレンダーの日詳細にスクショを添付 → OCRで自動読み取り or 手動入力 → アワード・スコアを集計とレーティングに反映）
- 1日の **最高 / 最低 / 平均** スコア表示
- **レーティング表示**（ダーツライブ換算の目安。直近30ゲーム＋今日のみの2種類）
- **アワードカウンター**（1日単位、自動判定＋手動 +/− 調整）
- 1日ごとの **メモ**
- **目標設定**（カウントアップ/クリケットCU の目標スコア、各カウンターの目標回数）
- **カレンダー**（目標達成日は ✓、一部達成は △。日タップで詳細・過去の修正）
- **グラフ**（平均・ベスト・MPR・各カウンターの推移を 14/30/90 日で表示）
- バックアップのエクスポート / インポート

## 自動カウントのルール

| カウンター | 自動判定 |
|---|---|
| ハットトリック | 1ラウンドでブル3本（両ゲーム） |
| BLACK | 1ラウンドで D-BULL×3 → **ハットトリックにも +1** |
| 9マーク | クリケットCUで1ラウンド9マーク |
| 変則7マーク | 手動カウントのみ（ホーム画面の +/−） |
| Bull miss T20 | 手動カウントのみ（ホーム画面の +/−） |
| T15〜T20 BED | 1ラウンドで同じトリプル3本（両ゲーム） |

すべてのカウンターはホーム画面（今日）またはカレンダーの日詳細（過去日）から手動調整できます。
手動で BLACK を +1 した場合もハットトリックに +1 されます。

## レーティングについて

ダーツライブの換算目安です（ファットブル基準）。

- 01側: カウントアップ平均 ÷ 8 = PPR。Rt.2 = 40.00 から 20/3 刻み（Rt.10 ≒ カウントアップ平均 747）
- クリケット側: MPR。Rt.2 = 1.30 から 0.2 刻み
- 総合 Rt = 両方の平均（四捨五入）。フライト: C / CC / B / BB / A / AA / SA

実際のダーツライブレーティングは直近のゲーム内容から独自に算出されるため、あくまで参考値です。

## ダーツライブ記録の取り込み方

1. カレンダー（または履歴）でその日をタップ → 「ダーツライブの記録」カード
2. **📷 スクショ追加** でダーツライブアプリのアワード画面・スコア画面のスクリーンショットを添付（画像は端末内の IndexedDB に縮小保存）
3. **🔍 画像から読み取る** で OCR（自動文字認識）を実行 → 確認フォームに候補が入る
   - 初回はネット接続が必要（認識エンジンをダウンロードするため）。精度は画像により差があるので、**必ず実際の記録と見比べて修正**してから反映してください
   - うまく読めない場合は **✏️ 手動で入力** からそのまま入力できます
4. **反映する** と、アワードはその日のカウンターに、スコアは「DL」バッジ付きのゲームとして最高/最低/平均・レーティングに反映されます
   - 反映は「その日のダーツライブ記録の上書き」なので、何度やり直しても二重計上にはなりません
   - クリケットCUの取り込みスコアはマーク数が不明なため、MPR の計算には含めません

## PC で使う

`index.html` を Chrome で開くだけで動きます（データはブラウザ内に保存）。

## Android スマホで使う（推奨手順）

PWA はインターネット上に置く（ホスティングする）必要があります。無料の GitHub Pages が簡単です。

1. https://github.com でアカウントを作る（無料）
2. 「New repository」でリポジトリを作る（例: `darts-app`、Public）
3. 「uploading an existing file」からこのフォルダの中身を全部ドラッグ＆ドロップしてアップロード
4. リポジトリの Settings → Pages → Branch を `main` にして Save
5. 数分後に表示される URL（`https://ユーザー名.github.io/darts-app/`）をスマホの Chrome で開く
6. Chrome のメニュー（⋮）→「ホーム画面に追加」→「インストール」

これでホーム画面のアイコンから普通のアプリと同じように起動でき、オフラインでも動きます。

> Netlify（netlify.com の「Drop」にフォルダをドラッグするだけ）でも同様に公開できます。

## データの保存とバックアップ

- データは **端末のブラウザ内**（localStorage）に保存されます。サーバーには送信されません。
- 機種変更やブラウザのデータ消去でデータが消えるため、**設定 → バックアップをダウンロード** で定期的に JSON を保存してください（Google Drive などに置くと安心）。
- 復元は **設定 → バックアップから復元** で JSON ファイルを選びます。
